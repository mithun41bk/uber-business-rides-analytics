 Data science project analyzing 2016 Uber ride logs with end‑to‑end workflow: data loading, cleaning, feature engineering (date, time, day‑night, round trips, weekdays, ride durations), exploratory data analysis, and business insights. The notebook explores ride demand by time of day and weekday, monthly miles......

Executive Summary with key findings

Dataset overview and preprocessing methodology

Exploratory Data Analysis (EDA) with statistical summaries

5 machine learning models (Random Forest, Logistic Regression, K-Means, ARIMA, Isolation Forest)

Business insights and cost optimization strategies

Statistical hypothesis testing (T-tests, Chi-square tests)

Complete code implementation details and explanations

Technical specifications and reproducibility guidelines

15 references to academic sources and tools

Key Content:

Data Understanding: 1,155 rides, 12,204.7 miles, 93.3% business focus

Temporal Patterns: Peak hours identified (9 AM, 2 PM, 5 PM)

Predictive Models: Distance prediction (R²=0.528), Category classification (94.2% accuracy)

Business Recommendations: $1,500-5,500 annual savings opportunities

Statistical Validation: Confirmed significant differences in business vs. personal travel
